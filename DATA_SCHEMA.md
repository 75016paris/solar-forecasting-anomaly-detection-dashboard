# Data schema notes

Current local data uses project-shaped CSV values with visible company/client names, plant labels and plant IDs replaced by neutral demo identifiers.

> Privacy note: this is **name-redacted operational-style data**, not fully synthetic data. It is included only to demonstrate the forecasting/anomaly-detection workflow.

## Publication status by path

| Path | Current role | Public status |
|---|---|---|
| `data/inverter_plants.csv` | Demo plant labels and capacities | Names/addresses and plant IDs redacted |
| `data/inverter_five_minutes_generation_logs.csv` | Five-minute generation time series | Uses neutral plant IDs and no company/client names |
| `data/inverter_hourly_generation_logs.csv` | Hourly inverter reference source | Uses neutral plant IDs and no company/client names |
| `data/inverter_daily_generation_logs.csv` | Daily inverter reference source | Uses neutral plant IDs and no company/client names |
| `data/plants_meter_data_logs_hourly.csv` | Hourly meter reference source | Plant/meter IDs and GUIDs remapped |
| `data/plants_billing_meter_logs.csv` | Billing meter reference source | Plant/meter IDs and GUIDs remapped |
| `open_data/gazipur_weather.csv` | Weather history used by the app | Location label generalized |
| `clean/*.csv` | Generated cleaned pipeline outputs | Derived from the name-redacted local data |
| `models/*` | Saved model artifacts and metrics | Derived from the name-redacted local data |

## Raw input files

### `data/inverter_plants.csv`

| Column | Meaning |
|---|---|
| `plant_id` | Neutral demo identifier used to join generation rows |
| `plant_name` | Neutral plant label |
| `plant_address` | Generalized location label |
| `plant_type` | Generalized plant category |
| `plant_capacity` | DC capacity in kWp |
| `system_type` | Generalized system configuration label |
| `inverter_provider` | Generalized provider label |
| `status` | Plant status |

### `data/inverter_five_minutes_generation_logs.csv`

| Column | Meaning |
|---|---|
| `plant_id` | Neutral demo identifier matching `inverter_plants.csv` |
| `generation_date` | UTC timestamp at five-minute resolution |
| `generation_amount` | Wh generated during the interval |

### Reference source CSVs

These files are used for validation/source-agreement diagnostics. They are not blindly merged into the hourly model target.

| File | Meaning |
|---|---|
| `data/inverter_hourly_generation_logs.csv` | Hourly inverter production reference |
| `data/inverter_daily_generation_logs.csv` | Daily inverter production reference |
| `data/plants_meter_data_logs_hourly.csv` | Cumulative hourly meter readings with remapped meter IDs/guids |
| `data/plants_billing_meter_logs.csv` | Cumulative billing meter readings with remapped meter IDs/guids |

### `open_data/gazipur_weather.csv`

OpenWeather-shaped hourly weather rows. Important columns used by the app include:

| Column | Meaning |
|---|---|
| `dt_iso` | UTC timestamp string |
| `temp` | Temperature in Celsius |
| `visibility` | Visibility proxy |
| `rain_1h` | One-hour rain volume/proxy |
| `clouds_all` | Cloud cover percentage |
| `humidity` | Relative humidity percentage |
| `wind_speed` | Wind speed proxy |

## Cleaned/generated files

Generated by:

```bash
python3 build_clean_dataset.py
```

| File | Meaning |
|---|---|
| `clean/plants_clean.csv` | Five selected plant metadata rows |
| `clean/generation_5min_clean.csv` | Cleaned/deduplicated selected-plant five-minute inverter records |
| `clean/hourly_model_targets.csv` | Hourly modeling targets from five-minute data; expected records/hour = 12; incomplete hours are `NaN` targets |
| `clean/daily_source_agreement.csv` | Daily diagnostics comparing five-minute inverter totals with hourly/daily inverter and meter references |
| `clean/weather_hourly_clean.csv` | Hourly weather features used by loaders/training when present |
| `clean/data_quality_summary.csv` | Per-plant completeness/source-agreement summary |

## Model artifacts

Saved model artifacts per plant:

- model `.pkl` file for the selected best model;
- `feature_columns.pkl`;
- `plant_info.pkl`;
- `model_comparison.csv`.

Current model metrics are based on cleaned hourly targets: incomplete hourly targets are excluded, and plant-days with strong daily inverter disagreement can be excluded from training/evaluation.

## Regeneration commands

```bash
python3 build_clean_dataset.py
python3 train_all_models.py
```
